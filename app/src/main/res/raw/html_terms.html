Tezov Bank Application

L'application "Tezov Bank" (dénommé cette "App") présente une application de démonstration. Toutes les données présentées sont factices. Cette App ne vend rien et ne collecte aucunes données. Elle est totalement gratuite d'utilisation et est la propriété de Tezov.


A savoir:

  • Tous les logins et mots de passes sont acceptés. Il suffit de mettre 4 chiffres.
  • Toutes les pages ne sont pas implémentées. Quand il n'y a pas de contenu, une snackbar vous en informe.

Vous pouvez parcourir la suite de ce document pour avoir des informations supplémentaires.

En savoir plus

Cette application est réalisé majoritairement en Koltin. L'interface visuel est 100% réalisé avec Compose.

  • Architecture: 3 modules (core - 100% kotlin, core - android UI, application - UI)
  • Injection de dépendance avec Dagger multi-module (State et Action fourni à l'UI)
  • Séparation UI avec injection du State (donnée + etats de la vue) et Action (gestion action d'utilisateur)
  • Navigation en seule source de vérité (coordinator pattern)
  • Chaque module a un module de Test associé.
  • La totalité du contenu affiché par l'ui est reçu par le viewModel (Sauf bottom navigation, top app bar et icon/image). La vue se construit en fonction des données reçu du viewModel

En cours d'ajout:

  • un module qui contiendra le viewModel
  • un module qui contiendra la logique server
  • un module qui contiendra les fonctionnaliteés propre à la plateforme (réseaux, persistance de données, camera, etc...)

Le tout organisé avec une logique de clean architecture.

UI

Liste des composants UI que l'on trouve dans cette application

  • bottom navigation bar
  • top navigation bar
  • dialog
  • bottom sheet
  • snackbar
  • tablayout
  • pager (full screen et carousel)
  • collaspible animated header (Compose animation + Motion Layout + Custom Layout)
  • drop down menu
  • vue custom cf. login keyboard

Le theming est intégralement réalisé en Kotlin pour compose. Un theme générale est founi à tous les composants peu importe le niveau de héirachie. Chaque composant peut customiser son propre style. L'architecture du theme est fournie par le module core - android UI qui complémente le materialTheme et apporte une notion d'états (default, success, error, alert, ...) d'un composant affectant le jeu de couleur/graisse/dimension/etc.
